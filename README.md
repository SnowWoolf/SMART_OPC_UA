# SMART_OPC_UA

## Установка/обновление одной командой:
```
curl -fsSL https://raw.githubusercontent.com/SnowWoolf/SMART_OPC_UA/main/um_opcua/deploy.sh | bash
```
Сервис запустится под именем `um_opcua`

### Конфигурационные файлы

#### /opt/um_opcua/config/API.cfg

сетевые настройки API-сервера

По-умолчанию:
```
URL=http://localhost
LOGIN=admin
PASSWORD=admin
PROTOCOL=40
```
#### /opt/um_opcua/config/tags.csv

Таблица тегов

| Тип устройства | Канал | Тип показаний | Тег API | Тег SCADA | Тип тега | Тип данных |
| ----------- | ----------- | ----------- | ----------- | ----------- | ----------- |----------- |

- Тип устройства - кодовый *номер* устройства из таблицы ПУ УМ (`"type":`)
- Канал - номер канала многоканального ПУ. **Для одноканальных указать 0**
- Тип показаний - `"measure":` в строке JSON запроса веб УМ
- Тег API - `"tag":` в строке JSON запроса веб УМ
- Тег SCADA - имя тега которое будет отображаться в дереве объектов
- Тип тега - Определяет, как OPC-сервер должен работать с тегом (текущие показания или исторические)
- Тип данных - Определяет, какого типа данные у тега (число, строка, дата, ...)
 
---
### Пояснения к новому tags.csv

`channel` (Канал)

Номер канала многоканального ПУ (для теплового счетчика)

Допустимые значения:

- `channel=0` — для неканальных тегов

- `channel=1`, `2` — для архивов/текущих данных по каналу


`Kind` (Тип тега)

Определяет, как OPC-сервер должен работать с тегом.

Допустимые значения:

- `current` — текущее значение, читается обычным OPC UA Read

- `history` — исторический параметр, читается через OPC UA HistoryRead

Примеры:

- напряжение, ток, текущее время → `current`

- энергия на начало суток, энергия на начало месяца, профиль мощности → `history`

`ValueType` (Тип данных)

Определяет, какого типа данные у тега.

Допустимые значения:

- `double` — обычное числовое значение

- `datetime` — дата и время

Примеры:

- напряжение, ток, энергия, мощность → `double`

- время счетчика (`GetTime`) → `datetime`

## Сборка deb-пакета
Собирать на УМ!

Скачать на УМ `build_deb.sh` и выполнить:

```
chmod +x build_deb.sh
PKG_VERSION=1.0.0 ./build_deb.sh
```
`PKG_VERSION=` - указать актуальную версию для пакета

Установка:
`sudo dpkg -i имя_пакета.deb`

Удаление:
`sudo dpkg -r um-opcua` (конфиги останутся)
`sudo dpkg --purge um-opcua` (удаление вместе с конфигами)
`sudo apt remove um-opcua` (удаление вместе с зависимостями)
